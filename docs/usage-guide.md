# 產品使用說明

本文件整理 Medi Check Frontend 的主要產品操作守則與使用範圍。

## 登入與註冊

- 使用者可透過註冊頁建立帳號。
- 登入後會進入受保護頁面。
- 前端會使用登入狀態與 token 呼叫後端 API。

## 首頁與提醒事件

- 首頁會顯示當日或近期提醒事件。
- 可快速確認接下來要服用的藥品與時段。
- 首頁內容會依目前登入者與可見病人情境而不同。

## 病人管理

- 可查看病人列表。
- 支援新增、編輯與檢視病人資料。
- 病人資料會作為藥品、提醒排程與服藥紀錄的關聯基礎。

## 藥品管理

- 可查看藥品列表。
- 支援搜尋、劑型篩選、排序與分頁。
- 列表項目會顯示藥品名稱與對應病人。
- 新增藥品採 step flow：
  - 選擇病人
  - 填寫藥品資料
- 編輯藥品流程會先確認病人，再進入藥品資訊編輯。

## 提醒排程管理

- 可新增、檢視與編輯提醒排程。
- 新增提醒採 step flow：
  - 選擇病人
  - 選擇藥品
  - 設定提醒內容
- 目前流程支援：
  - 病人搜尋與分頁
  - 藥品搜尋
  - 劑量、頻率、結束條件設定
  - 時段與提醒規則設定

## 服藥紀錄

- 可查看歷史紀錄。
- 可編輯特定紀錄內容。
- 可搭配病人、日期等條件檢視。

## 邀請與照護關係管理

- 可發送邀請。
- 可查看收到的邀請。
- 可查看已發出的邀請。
- 可接受、拒絕或撤銷邀請。
- 邀請卡片通常會顯示：
  - 邀請對象或邀請者
  - 邀請用途
  - 權限
  - 邀請日期
  - 狀態
